---
title: "Academics API"
description: "Manage departments, programs, classes, sections, subjects, teacher-subject mappings, and timetables."
---

<Info>
  All endpoints require JWT authentication via the `Authorization: Bearer <token>` header or the `access_token` cookie.
</Info>

## Resources

- **Departments** — top-level organizational units (separate `/api/department/` prefix)
- **Programs** — academic programs within a department (e.g. BS Computer Science)
- **Classes** — years or semesters within a program
- **Sections** — class subdivisions (A, B, C …)
- **Subjects** — individual courses within a class
- **TSM (Teacher-Subject Mapping)** — assigns a teacher to a subject
- **TCM (Teacher-Class-Section Mapping)** — assigns a teacher to a class/section
- **Timetable** — per-class schedule entries

---

## Departments

Departments are managed under the `/api/department/` prefix.

### GET /api/department/get

Return all departments with full details.

```bash
curl http://localhost:3003/api/department/get \
  -H "Authorization: Bearer <access_token>"
```

### GET /api/department/get/names

Return a lightweight list of department IDs and names.

### POST /api/department/create

<ParamField body="name" type="string" required>
  Department name (must be unique).
</ParamField>

<ParamField body="description" type="string">
  Optional description.
</ParamField>

<ParamField body="hodId" type="number">
  Staff ID of the head of department.
</ParamField>

```bash
curl -X POST http://localhost:3003/api/department/create \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Computer Science", "description": "CS Department"}'
```

### PATCH /api/department/update

<ParamField query="depID" type="number" required>
  Department ID.
</ParamField>

### DELETE /api/department/remove

<ParamField query="depID" type="number" required>
  Department ID.
</ParamField>

## Programs

### GET /api/academics/program/get/all

Return all academic programs with full details.

```bash
curl http://localhost:3003/api/academics/program/get/all \
  -H "Authorization: Bearer <access_token>"
```

### GET /api/academics/program/get/all/names

Return a lightweight list of program IDs and names.

### POST /api/academics/program/create

<ParamField body="name" type="string" required>
  Program name.
</ParamField>

<ParamField body="level" type="string" required>
  Academic level. Must be one of: `INTERMEDIATE`, `UNDERGRADUATE`, `COACHING`, `DIPLOMA`, `SHORT_COURSE`.
</ParamField>

<ParamField body="departmentId" type="string | number" required>
  ID of the department that owns this program.
</ParamField>

<ParamField body="duration" type="string" required>
  Program duration (e.g. `4 years`).
</ParamField>

<ParamField body="description" type="string">
  Optional description.
</ParamField>

<ParamField body="rollPrefix" type="string">
  Prefix used when auto-generating roll numbers.
</ParamField>

```bash
curl -X POST http://localhost:3003/api/academics/program/create \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "BS Computer Science", "level": "UNDERGRADUATE", "departmentId": 1, "duration": "4 years"}'
```

### PATCH /api/academics/program/update

<ParamField query="programID" type="number" required>
  ID of the program to update.
</ParamField>

Accepts the same fields as the create endpoint, all optional.

### DELETE /api/academics/program/remove

<ParamField query="programID" type="number" required>
  ID of the program to delete.
</ParamField>
