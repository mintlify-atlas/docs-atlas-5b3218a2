## Inquiries

An inquiry records a prospective student's interest in admission.

### GET /api/front-office/get/inquiries

<ParamField query="programId" type="number">
  Filter by program of interest.
</ParamField>

<ParamField query="page" type="number">
  Page number. Defaults to `1`.
</ParamField>

<ParamField query="limit" type="number">
  Results per page. Defaults to `10`.
</ParamField>

```bash
curl "http://localhost:3003/api/front-office/get/inquiries?page=1&limit=20" \
  -H "Authorization: Bearer <access_token>"
```

### POST /api/front-office/create/inquiry

<ParamField body="studentName" type="string" required>
  Prospective student's name.
</ParamField>

<ParamField body="fatherName" type="string" required>
  Father's name.
</ParamField>

<ParamField body="contactNumber" type="string" required>
  Contact phone number.
</ParamField>

<ParamField body="programInterest" type="string" required>
  Program the student is interested in.
</ParamField>

<ParamField body="fatherCnic" type="string">
  Father's CNIC number.
</ParamField>

<ParamField body="email" type="string">
  Email address.
</ParamField>

<ParamField body="address" type="string">
  Home address.
</ParamField>

<ParamField body="previousInstitute" type="string">
  Name of the previous school or college.
</ParamField>

<ParamField body="remarks" type="string">
  Staff remarks.
</ParamField>

<ParamField body="status" type="string">
  Inquiry status: `NEW`, `APPROVED`, `REJECTED`, or `FOLLOW_UP`.
</ParamField>

```bash
curl -X POST http://localhost:3003/api/front-office/create/inquiry \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "studentName": "Usman Ali",
    "fatherName": "Ali Raza",
    "contactNumber": "03009876543",
    "programInterest": "BS Computer Science"
  }'
```

### PATCH /api/front-office/update/inquiry

<ParamField query="id" type="number" required>
  Inquiry ID.
</ParamField>

Accepts any subset of the create fields.

### DELETE /api/front-office/delete/inquiry

<ParamField query="id" type="number" required>
  Inquiry ID.
</ParamField>

### PATCH /api/front-office/rollback/inquiry

Reset an inquiry's status to `NEW`.

<ParamField query="id" type="number" required>
  Inquiry ID.
</ParamField>

</ParamField>

<ParamField body="complainantName" type="string" required>

</ParamField>

</ParamField>

  Complaint subject or title.
</ParamField>

  Full complaint details.
</ParamField>

<ParamField body="status" type="string" required>
  Current status (e.g. `OPEN`, `IN_PROGRESS`, `RESOLVED`).
</ParamField>

  ID of the staff member responsible for resolution.
</ParamField>

```bash
curl -X POST http://localhost:3003/api/front-office/create/complaint \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "Facility",
    "complainantName": "Mr. Hassan",
    "contact": 3001234567,
    "subject": "Broken chairs",
    "details": "Chairs in room 101 need replacement",
    "status": "OPEN"
  }'
```
