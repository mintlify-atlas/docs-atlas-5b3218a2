## Institute settings

### GET /api/configuration/institute-settings

Return the current institute configuration (name, address, logo URL, contact details, etc.).

```bash
curl http://localhost:3003/api/configuration/institute-settings \
  -H "Authorization: Bearer <access_token>"
```

<ResponseField name="id" type="number">
  Settings record ID.
</ResponseField>

<ResponseField name="instituteName" type="string">
  Name of the institution.
</ResponseField>

<ResponseField name="address" type="string">
  Physical address.
</ResponseField>

<ResponseField name="phone" type="string">
  Contact phone number.
</ResponseField>

<ResponseField name="email" type="string">
  Contact email.
</ResponseField>

<ResponseField name="logoUrl" type="string">
  URL of the institution logo.
</ResponseField>

### PATCH /api/configuration/institute-settings

Update any institute setting. All fields are optional; only the provided fields are updated.

```bash
curl -X PATCH http://localhost:3003/api/configuration/institute-settings \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"instituteName": "Concordia PSH", "phone": "+92-51-1234567"}'
```

## Report card templates

Templates control the visual layout of generated student report cards.

### POST /api/configuration/report-card-templates

Create a new report card template.

<Note>
  The template body contains layout configuration. Refer to `CreateReportCardTemplateDto` in the source for the exact schema.
</Note>

```bash
curl -X POST http://localhost:3003/api/configuration/report-card-templates \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"name": "Standard Report Card", "isDefault": true, "layout": {}}'
```

### GET /api/configuration/report-card-templates

List all report card templates.

### GET /api/configuration/report-card-templates/default

Return the template currently marked as default.

```bash
curl http://localhost:3003/api/configuration/report-card-templates/default \
  -H "Authorization: Bearer <access_token>"
```
