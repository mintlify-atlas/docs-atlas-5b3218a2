## Overview

The Finance module provides a consolidated view of the institution's financial activity. It combines manually entered income and expense records with data automatically pulled from fee challans (as income), payroll payments (as salary expenses), hostel expenses, and inventory purchases and maintenance costs. Admins can close periods (daily, weekly, or monthly) and store balance records for reporting.

<Note>
  Paid fee challans automatically appear under the **Fee** income category. Paid payroll records automatically appear under the **Salaries** expense category. Hostel expenses and inventory purchases appear under **Hostel** and **Inventory** categories respectively.
</Note>

## Workflows

<Tabs>
  <Tab title="Record income">
    <Steps>
      <Step title="Navigate to Finance → Income">
        Open the income list. Existing income entries including automatic fee payment entries are displayed.
      </Step>
      <Step title="Add a new entry">
        Click add. Enter the date, category (e.g., Donations, Grants, Rental), description, and amount.
      </Step>
      <Step title="Save">
        The entry is stored as a `FinanceIncome` record. It will appear in the income total.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Record expense">
    <Steps>
      <Step title="Navigate to Finance → Expenses">
        Open the expense list. Payroll, hostel, and inventory expenses appear automatically alongside manual entries.
      </Step>
      <Step title="Add a new entry">
        Click add. Enter the date, category (e.g., Utilities, Rent, Supplies), description, and amount.
      </Step>
      <Step title="Save">
        The entry is stored as a `FinanceExpense` record.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Close a period">
    <Steps>
      <Step title="Navigate to Finance → Closing">
        Open the closing records list.
      </Step>
      <Step title="Create a closing entry">
        Enter the date, type (daily, weekly, or monthly), total income, total expense, and net balance. Add optional remarks.
      </Step>
      <Step title="Save">
        The closing record is stored as a `FinanceClosing` entry for audit and reporting.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Data fields

<AccordionGroup>
  <Accordion title="Income record">
    | Field | Description |
    |-------|-------------|
    | `date` | Date of the income |
    | `category` | Category label (e.g., `Fee`, `Donation`, `Rental`) |
    | `description` | Description of the income source |
    | `amount` | Amount received |
  </Accordion>
  <Accordion title="Expense record">
    | Field | Description |
    |-------|-------------|
    | `date` | Date of the expense |
    | `category` | Category label (e.g., `Salaries`, `Utilities`, `Hostel`, `Inventory`) |
    | `description` | Description of the expense |
    | `amount` | Amount spent |
  </Accordion>
  <Accordion title="Closing record">
    | Field | Description |
    |-------|-------------|
    | `date` | Closing date |
    | `type` | Period type (e.g., `daily`, `weekly`, `monthly`) |
    | `totalIncome` | Total income for the period |
    | `totalExpense` | Total expense for the period |
    | `netBalance` | `totalIncome - totalExpense` |
    | `remarks` | Optional notes |
  </Accordion>
</AccordionGroup>

## Automatic expense categories

| Source | Category | Trigger |
|--------|----------|---------|
| Paid fee challans | Fee | `status = PAID` on `FeeChallan` |
| Paid payrolls | Salaries | `status = PAID` on `Payroll` |
| Hostel expense records | Hostel | Any `HostelExpense` entry |
| Inventory purchases | Inventory | Any `SchoolInventory` entry (`totalValue`) |
| Inventory maintenance | Inventory | Any `InventoryExpense` entry |

## Permissions

This module requires the `Finance` permission in `admin.permissions.modules`.
