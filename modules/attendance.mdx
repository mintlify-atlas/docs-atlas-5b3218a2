## Overview

The Attendance module handles both student and staff attendance. Student attendance is tracked per subject per day, meaning a student can have multiple attendance records on the same day (one per subject). Staff attendance is one record per staff member per day. The system respects holidays and weekends, blocking attendance marking on non-working days.

## Key features

<CardGroup cols={2}>
  <Card title="Student attendance marking" icon="user-check">
    Mark attendance for a class or section on a specific date for a selected subject. Status options: PRESENT, ABSENT, LEAVE, HALF_DAY.
  </Card>
  <Card title="Auto-generation" icon="rotate">
    Trigger bulk attendance generation for a date. The system creates PRESENT records for all active students across all subjects, skipping records that already exist.
  </Card>
  <Card title="Staff attendance" icon="users-between-lines">
    Mark daily attendance for all active teaching and non-teaching staff. Auto-generation creates PRESENT records for all active staff on a given date.
  </Card>
  <Card title="Leave integration" icon="calendar-xmark">
    When a student leave request is approved, attendance records for all affected past and current dates are automatically upserted with status LEAVE (or ABSENT if rejected).
  </Card>
  <Card title="Holiday awareness" icon="umbrella-beach">
    The system checks for holidays (exact date matches and repeating yearly holidays) and weekends before allowing attendance to be marked. Sundays are always blocked.
  </Card>
  <Card title="Attendance reports" icon="chart-bar">
    Generate attendance reports for a date range, filtered by class and section. Results show each student's attendance status per subject per day.
  </Card>
</CardGroup>

## Workflows

<Tabs>
  <Tab title="Mark student attendance">
    <Steps>
      <Step title="Select class, section, subject, and date">
        Choose the class, optionally a section, the subject for which attendance is being marked, and the date. The system checks if the date is a weekend or holiday.
      </Step>
      <Step title="Review pre-filled attendance">
        The system loads all students for the selected class/section with any existing attendance records for that date and subject.
      </Step>
      <Step title="Update statuses">
        Change individual student statuses to PRESENT, ABSENT, LEAVE, or HALF_DAY.
      </Step>
      <Step title="Save">
        Submit the attendance. Existing records are updated; new records are created for students who had no prior record.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Auto-generate attendance">
    <Steps>
      <Step title="Trigger generation">
        Call the generate endpoint with a date and type (`student` or `teacher`). The system validates that the date is not a holiday or weekend.
      </Step>
      <Step title="Leave check">
        The system fetches all approved or rejected leaves overlapping the target date and uses them to set initial statuses (LEAVE for approved, ABSENT for rejected).
      </Step>
      <Step title="Create records">
        Bulk attendance records are inserted for all active students or staff who do not already have a record for that date.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Staff attendance">
    <Steps>
      <Step title="Select date">
        Choose a date. The system validates it is not a weekend or holiday.
      </Step>
      <Step title="Mark attendance">
        Set the attendance status for each active staff member to PRESENT, ABSENT, LEAVE, or HALF_DAY.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Data fields

<AccordionGroup>
  <Accordion title="Student attendance record">
    | Field | Description |
    |-------|-------------|
    | `studentId` | Linked student |
    | `classId` | Class at time of marking |
    | `sectionId` | Section (null for direct class students) |
    | `subjectId` | Subject the attendance is for |
    | `teacherId` | Teacher who marked attendance (optional) |
    | `date` | Date of attendance (stored as DATE, no time component) |
    | `status` | `PRESENT`, `ABSENT`, `LEAVE`, `HALF_DAY`, or `HOLIDAY` |
    | `autoGenerated` | `true` if created by the bulk generation process |
    | `notes` | Optional notes (e.g., leave reason) |
  </Accordion>
  <Accordion title="Staff attendance record">
    | Field | Description |
    |-------|-------------|
    | `staffId` | Linked staff member |
    | `date` | Date of attendance |
    | `status` | `PRESENT`, `ABSENT`, `LEAVE`, `HALF_DAY`, or `HOLIDAY` |
    | `markedBy` | Admin who marked the attendance |
    | `autoGenerated` | `true` if created by bulk generation |
    | `notes` | Optional notes |
  </Accordion>
  <Accordion title="Holiday">
    | Field | Description |
    |-------|-------------|
    | `date` | Date of the holiday |
    | `title` | Holiday name |
    | `type` | Holiday type (e.g., `National`, `Religious`, `Gazetted`) |
    | `repeatYearly` | If `true`, the holiday applies to the same month/day every year |
    | `description` | Optional description |
  </Accordion>
</AccordionGroup>

## Permissions

This module requires the `attendance` permission in `admin.permissions.modules`. Teachers can also mark attendance for their own assigned classes through the teacher portal.
