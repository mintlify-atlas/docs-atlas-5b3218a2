## Prerequisites

Before you begin, make sure you have the following installed:

- [Docker](https://docs.docker.com/get-docker/) (v20.10 or later)
- [Docker Compose](https://docs.docker.com/compose/install/) (v2.0 or later)
- Git

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/concordia-psh.git
    cd concordia-psh
    ```
  </Step>

  <Step title="Configure environment variables">
    The `docker-compose.yml` already includes default values for local development. Copy the environment block below into a `.env` file if you want to override defaults, or proceed directly with the defaults.

    ```env
    # Database
    DATABASE_URL=mysql://root:root@mysql:3306/concordia

    # JWT secrets — change these before deploying to production
    JWT_ACCESS_SECRET=asdaskdpanqweqejlsadnasnd
    JWT_REFRESH_SECRET=09190199219jjkdjasjdnasnd

    # Application
    NODE_ENV=development
    PORT=3003
    ALLOWED_IP=http://localhost:5173

    # Cloudinary (file uploads)
    CLOUDINARY_URL=cloudinary://973774315816455:EV1VsJ4edk6mLdx1baGVJpa3dMc@dldigpb7f
    CLOUD_NAME=dldigpb7f
    CLOUDINARY_API_KEY=973774315816455
    CLOUDINARY_API_SECRET=EV1VsJ4edk6mLdx1baGVJpa3dMc
    ```

    <Warning>
      The JWT secrets and Cloudinary credentials above are sample values. Replace them with your own secrets before running in any environment accessible from the internet.
    </Warning>
  </Step>

  <Step title="Start all services">
    From the project root, run:

    ```bash
    docker compose up -d
    ```

    Docker Compose will start three services:

    | Service | Port | Description |
    |---|---|---|
    | `mysql` | — | MySQL 8.0 database (internal only) |
    | `backend` | `3003` | NestJS REST API |
    | `admin-panel` | `5173` | React admin panel |

    The backend waits for MySQL to pass its health check before starting, so the first launch may take 30–60 seconds.
  </Step>

  <Step title="Seed the database">
    After the backend is running, seed the initial super admin account:

    ```bash
    docker compose exec backend npx ts-node prisma/seed.ts
    ```

    This creates the default `SUPER_ADMIN` user defined in `prisma/seed.ts`.
  </Step>

  <Step title="Open the admin panel">
    Navigate to [http://localhost:5173](http://localhost:5173) in your browser.

    Log in with the super admin credentials created during the seed step. The `SUPER_ADMIN` role has unrestricted access to all modules.

    <Note>
      If you see a loading screen for more than a few seconds, the backend may still be initializing. Check the container logs with `docker compose logs backend`.
    </Note>
  </Step>
</Steps>
